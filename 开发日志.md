## 一、跨机器/跨 AI 接力指南（先读这段）

- **修改流程**：

  2. 改完 `node --check main.js` 过语法。
  3. 跑集成测试（见第四节），用真实妙记验证，**不要只靠肉眼**。
  4. 在本文件追加版本条目（改了什么/哪些方法/怎么验证）。
- **依赖**：`lark-cli`（飞书官方 CLI），实际路径 `~/.npm-global/bin/lark-cli`。插件 `detectCli()` 会自动探测，无需手填。需已 `lark-cli auth login` 且具备 minutes/vc:note/docx/docs.media 权限。
- **运行环境**：Node 22（`/opt/homebrew/opt/node@22/bin`）。Obsidian 渲染进程提供 `window`（`scheduleAutoSync` 用到 `window.setInterval`）。

## 二、这个插件干什么

定时轮询飞书妙记，把新妙记同步成 Obsidian 笔记。核心是抓 `note_doc_token` 对应的**飞书智能纪要 Docx**，里面已渲染好全部模块：总结、待办、智能章节、关键决策、其他决策、金句时刻、相关链接、图文(img)、图表/画板(whiteboard)。图片和画板会下载为本地附件。

## 三、关键技术约束（踩过的坑，别再踩）

1. **`docs +media-download` 不支持 `--format` flag**。传了会报 `unknown flag` 导致每次媒体下载崩溃。这是 0.2.0→0.3.0 修复的头号 bug。
2. **图片必须用 XML `<img src>`（持久 file_token）下载，不能用 href 直链**。markdown/XML 里图片的 `href` 是 `internal-api-drive-stream.feishu.cn/...?code=<authcode>` 直链，**authcode 几分钟就过期**（实测从 HTTP 200 变 400/0字节）。`src` 是真正的 file_token，走 `media-download --type media --token <src>` 持久可靠。
3. **画板用 `<whiteboard token>` + `media-download --type whiteboard`**，导出为 jpg。
4. **`media-download --output` 必须相对路径**（CLI 安全限制）。插件用 cwd=临时目录 + 相对 baseName，下载后读 buffer 再 `vault.adapter.writeBinary` 写进 vault。
5. **图片定位用 href 强匹配为主、文档下标为辅**。md 图片 URL === XML `<img href>`（同次抓取），用它当主键避免“串图”；下标兜底；数量不等时打 log 警告。
6. **`normalizeSmartMarkdown` 剥离**：`<title>`、`<readonly-block>`（含空占位/自闭合）、`<grid>`、`<column>`，并兜底清除残留 `<whiteboard>` 裸标签。
7. **无智能纪要的妙记**（纯练习录音、`note_doc_token` 为空）要优雅跳过，不报错。
